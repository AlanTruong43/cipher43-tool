# QUY TRÌNH VÀ PHƯƠNG THỨC CODE CHO DỰ ÁN AUTOMATION 43GPM

Tài liệu này hướng dẫn cách cấu trúc một script automation (ví dụ: `twitter.py`) để tương thích với hệ thống API Server và trình duyệt 43GPM.

**Automation engine: DrissionPage** (không dùng Selenium)

---

## 1. Cấu trúc cơ bản của một File Automation

Mọi script mới phải được đặt trong thư mục `project/` và có cấu trúc như sau:

```python
# -*- coding: utf-8 -*-
import encoding_fix  # BẮT BUỘC: Fix lỗi font chữ trên Windows

from DrissionPage import ChromiumPage, ChromiumOptions
import time

def run(profile_data):
    """
    Hàm entry point chính. API Server sẽ gọi hàm này.
    profile_data chứa: remote_debugging_address, profile_id, và các extra params.
    """
    debug_address = profile_data['remote_debugging_address']

    try:
        # Bước 1: Kết nối vào browser đang mở qua debug port
        co = ChromiumOptions().set_address(debug_address)
        page = ChromiumPage(co)

        # Bước 2: Chọn tab thực tế (bỏ qua tab extension)
        target_tab = None
        for tab in page.get_tabs():
            url = tab.url
            if not url.startswith("chrome-extension://") and not url.startswith("chrome://"):
                target_tab = tab
                break

        if target_tab is None:
            page.new_tab()
            target_tab = page.get_tab(page.tab_ids[-1])

        page.activate_tab(target_tab)

        # Bước 3: Thực hiện các bước thao tác
        # page.get("https://example.com")
        # el = page.ele('xpath://button[@id="submit"]', timeout=10)
        # el.click()

    except Exception as e:
        print(f">>> [ERROR] {e}")
    finally:
        print(">>> [INFO] Hoàn thành tác vụ.")
```

## 2. Các API DrissionPage hay dùng

### Kết nối browser
```python
co = ChromiumOptions().set_address("127.0.0.1:9222")
page = ChromiumPage(co)
```

### Điều hướng
```python
page.get("https://example.com")   # Mở URL
print(page.url)                    # Lấy URL hiện tại
page.back()                        # Quay lại
page.refresh()                     # Reload
```

### Tìm element (thay cho find_element / By)
```python
# Tìm 1 element — trả về None nếu không thấy (không throw exception)
el = page.ele('xpath://button[@id="submit"]', timeout=10)
el = page.ele('css:button.submit-btn', timeout=10)
el = page.ele('text:Xác nhận', timeout=10)          # Tìm theo text

# Tìm nhiều element
els = page.eles('xpath://input[@type="password"]')

# Tìm element con trong element cha
child = parent_el.ele('xpath:.//span')
```

### Tương tác element
```python
el.click()                          # Click thường
page.run_js("arguments[0].click();", el)   # JS click (dùng khi bị block)
el.input("nội dung", clear=True)    # Nhập text (tự clear trước)
el.clear()                          # Xóa nội dung
print(el.text)                      # Lấy text
print(el.attr("href"))              # Lấy attribute
```

### Chờ element (đợi hiển thị hoặc click được — khoanh vùng bằng ô vuông màu đỏ theo rule_coding.md)
```python
# DrissionPage tự chờ theo timeout — không cần WebDriverWait riêng
el = page.ele('xpath://div[@class="result"]', timeout=15)
if el:
    # element đã hiển thị, tiến hành thao tác
    el.click()
```

### Quản lý Tab
```python
tabs = page.get_tabs()              # Lấy danh sách tất cả tab
page.activate_tab(tab)             # Chuyển sang tab
page.new_tab("https://google.com") # Mở tab mới
page.close_tab(tab)                # Đóng tab
```

### Chạy JavaScript
```python
page.run_js("window.scrollTo(0, 500);")
result = page.run_js("return document.title;")
page.run_js("arguments[0].style.border='3px solid red';", el)  # Truyền element
```

### Iframe / Shadow DOM
```python
iframe = page.get_frame('xpath://iframe[@id="myFrame"]')
el_in_frame = iframe.ele('xpath://button')
```

## 3. Quy trình làm việc (Các bước thao tác)

Khi gửi yêu cầu tạo script mới, bạn chỉ cần cung cấp các thông tin sau:

1. **Tên Project**: Ví dụ: `facebook`, `tiktok`, `discord`.
2. **URL Mục tiêu**: Trang web cần truy cập đầu tiên.
3. **Các Bước Thao Tác (Sequence)**:
   - Bước 1: Kiểm tra xem đã login chưa (Dựa vào element nào).
   - Bước 2: Tìm nội dung/nút nào để tương tác.
   - Bước 3: Các hành động đặc biệt (Scroll, Wait, Highlighting).
4. **Phản hồi trực quan (Visual Feedback)**: Có cần hiện thông báo "Bot Connected" hay highlight nút xanh/đỏ không?

## 4. Các tiêu chuẩn kỹ thuật (Checklist dành cho AI)

Để đảm bảo script chạy ổn định, tôi (AI) sẽ luôn tuân thủ:

- **Encoding**: Luôn có `import encoding_fix` ở dòng đầu.
- **Connection**: Dùng `ChromiumOptions().set_address(debug_address)` để kết nối vào browser đang mở (không mở browser mới).
- **Prefix Logging**: Dùng prefix `>>> [TÊN]` để dễ trace trong console.
- **Tab Selection**: Luôn scan `page.get_tabs()` để chọn tab thực tế, bỏ qua `chrome-extension://` và `chrome://`.
- **Locator**: Xác định element bằng xpath, css selector — KHÔNG dùng tọa độ, hình ảnh hay text động.
- **Chờ element**: Dùng `page.ele(locator, timeout=N)` thay vì `time.sleep` cứng.
- **Visual Interaction**: Inject JavaScript để highlight element (đỏ khi lỗi, xanh khi thành công) hoặc hiện thông báo trên màn hình.
- **Fallback**: Nếu không tìm thấy element, thử các locator dự phòng trước khi raise exception.

## 7. Cách kích hoạt Automation (Endpoint)

Mọi script trong thư mục `project/` đều có thể được gọi thông qua trình duyệt hoặc phần mềm thứ 3 bằng URL:

**Cú pháp:**
`http://127.0.0.1:8000/execute/[Tên_Project]?port=[DEBUG_PORT]`

*Ví dụ:* Để chạy script `import_key_okx.py` ở port 9222:
`http://127.0.0.1:8000/execute/import_key_okx?port=9222&mnemonic=word1+word2&password=abc`

## 8. Ghi chú quan trọng

- Lấy debug port từ antidetect browser của bạn trước khi gọi API.
- Script sẽ tự động tìm tab đang mở hoặc mở tab mới tùy theo logic trong hàm `run()`.
- Dùng `page.run_js("arguments[0].click();", el)` khi tương tác với Extension để tránh bị crash.
- `import_key_okx_stealth.py` dùng Raw CDP (websocket) — không cần DrissionPage, giữ nguyên.
